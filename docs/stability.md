# Stability policy

candle is pre-1.0. This page tags each tool and resource as **stable** or
**experimental** so adopters know what they can rely on, and defines the
semver/deprecation rules that govern changes.

The source of truth for the tool list is the registration order in
[`internal/mcp/server.go`](../internal/mcp/server.go); for resources,
[`docs/resources.md`](resources.md).

## Tiers

- **Stable** — the output shape is committed. Breaking changes (removing or
  renaming a field, changing semantics) require a version bump per the rules
  below. Additive changes (new fields) are non-breaking and allowed anytime.
- **Experimental** — the shape may evolve before 1.0. Breaking changes are
  noted in the changelog but are not gated by semver. Adopters should not build
  hard dependencies on these without pinning a version.

## Versioning

candle follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html). The
changelog is [`CHANGELOG.md`](../CHANGELOG.md); GitHub release notes are
generated by GoReleaser from commit history.

- **Pre-1.0 (`0.x.y`)** — the project is MVP. Breaking changes to a *stable*
  tool/resource require a minor bump (`0.x`) and a changelog entry. Breaking
  changes to an *experimental* surface may happen in any release and are noted
  in the changelog. Additive field additions are non-breaking.
- **From `1.0.0`** — full semver. Breaking changes to *stable* surfaces require
  a major bump (`1.0` → `2.0`). Breaking changes to *experimental* surfaces are
  allowed in minor bumps (`1.x`) and noted in the changelog.

## Deprecation

- A deprecated tool or resource is marked in [`docs/tools.md`](tools.md) (or
  [`docs/resources.md`](resources.md)) and the changelog, with a replacement if
  one exists.
- A deprecated surface is removed only in the next major release, and only after
  at least one release carrying the deprecation notice.
- While deprecated, the surface continues to work unchanged.

## Promoting experimental → stable

An experimental surface is promoted to stable when its output shape has held
through a release cycle without forcing a breaking change. The promotion is
noted in the changelog and the table above is updated in the same release.
